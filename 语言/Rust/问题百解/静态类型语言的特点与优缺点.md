1. 定义

- 静态类型语言在编译时确定每个变量、表达式和函数的类型。类型信息在程序运行前就已存在并被检查。

2. 主要特征

- 编译期类型检查：编译器会在编译阶段验证类型一致性，发现类型不匹配、调用不存在的方法、错误的参数类型等问题并报错。
- 类型显式或推断：有些静态语言要求显式声明类型（如早期的C、Java），有些支持类型推断（如Rust、TypeScript、Kotlin），可在不写出具体类型的情况下由编译器推断。
- 强类型/弱类型区别：静态类型不等于强类型，但很多静态语言同时也是强类型（例如Rust是强类型，**禁止隐式类型转换**）。

3. 优点

- 早期发现错误：类型错误在编译时被捕获，减少运行时异常。
- 更好的性能：编译器能据类型进行优化（内存布局、内联、去虚拟化），通常比动态语言更高效。
- 可维护性与可读性：明确的类型契约帮助理解接口和数据结构，利于团队协作。
- 更好的工具支持：类型信息增强了IDE自动补全、重构、静态分析能力。
- 安全性：像Rust这种静态语言还能借助类型系统和所有权/借用规则提供内存安全和并发安全保证。

4. 缺点

- 开发初期可能较繁琐：需要定义类型或适应类型系统（尽管类型推断缓解了这一点）。
- 学习曲线：复杂的类型系统（泛型、生命周期、类型约束）可能增加学习难度。
- 某些快速原型或动态行为表达起来不如动态语言灵活。

5. 与动态类型语言的对比（简要）

- 静态类型：类型在编译期确定，出错更早，性能与工具支持更好。
- 动态类型：类型在运行时决定，更灵活，适合快速原型，但可能在运行时出现类型错误。

6. 与Rust相关的补充

- Rust是静态且强类型的语言，同时提供完善的类型推断（let x = 5; 编译器推断为 i32 或 i64），拥有所有权（ownership）、借用（borrowing）与生命周期（lifetimes）机制，用类型系统在编译期保证内存与并发安全。
- 典型优势体现在系统编程、性能敏感与并发场景，同时降低运行时错误与内存错误的风险。
